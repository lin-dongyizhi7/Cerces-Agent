# Cerces-Agent

针对昇腾NPU分布式训练监测系统，我需要实现数据采集端和处理端（center端）。下面先考虑center端的实现。

在center端，我需要实现一个这样的系统，它的层级结构如下。

1.通信转义层。由C++实现的接收来自多机的流式监控数据，使用protobuf协议，并实时转义为结构化数据发送给下一层。

2.异常检测层。实时接收来自通信转义层的结构化数据，进行异常检测和预警。同时实现多种异常检测方法，实现对应的类，通过参数选择调用的异常检测方法（可以同时调用多种方法进行比较），考虑用C++实现还是python实现哪种更好。异常的判定包括统计学的3σ、对比rank和历史迭代的规则，以及其他经典异常检测算法。

3.根因分析层。对异常时数据进行根因分析，实现因果图谱和图神经网络等多种根因分析方法。

4.可视化层。首先与通信转义层关联，将监控数据实时展示，绘制动态图线，可以在控制端选择要显示的指标。同时与异常检测层和根因分析层关联，进行异常报警显示与根因分析结果展示，根因分析中显示loading。可视化层使用python结合web开发实现，考虑使用原生html/css/js以提高性能或使用开发框架。

参考该需求以及昇腾NPU分布式训练监测指标体系v2.0.md，给出完整的需求及设计文档Require.md和Design.md。
